# SaaS UI Component Library — 페이지별 기능 설명서

---

## 전체 페이지 구성

```
┌─────────────────────────────────────────────┐
│  Navbar (모든 페이지 공통)                      │
│  로고 · 탐색 링크 · 검색(⌘K) · GitHub         │
├─────────────────────────────────────────────┤
│                                             │
│           각 페이지 콘텐츠 영역               │
│                                             │
├─────────────────────────────────────────────┤
│  Toaster (알림, 우측 상단)                     │
└─────────────────────────────────────────────┘
```

총 **3개 페이지**, 모든 페이지는 Navbar 아래에 렌더링된다.

| 페이지 | URL | 한줄 설명 |
|--------|-----|-----------|
| 홈 | `/` | 소개 + 추천 컴포넌트 + 카테고리 |
| 탐색 | `/explore` | 전체 컴포넌트 브라우징 |
| 상세 | `/components/:slug` | 라이브 데모 + 코드 복사 |

---

## 공통 — Navbar

모든 페이지 상단에 고정되는 내비게이션 바.

| 요소 | 동작 |
|------|------|
| 로고 (SaaS UI) | 클릭 → 홈(`/`)으로 이동 |
| 탐색 링크 | 클릭 → `/explore`로 이동 |
| 검색 버튼 | 클릭 또는 `⌘K` → 검색 모달 오픈 |
| GitHub 아이콘 | 외부 링크 |

### 검색 동작
1. `⌘K` 또는 검색 버튼 클릭 → 모달 오픈
2. 검색어 입력 후 Enter
3. `/explore?q=검색어`로 이동, Supabase FTS로 결과 조회
4. `ESC` 또는 배경 클릭 → 모달 닫기

## 1. 홈 페이지 (`/`)

사이트 첫 화면. 프로젝트를 소개하고 컴포넌트로 유도한다.

```
┌─────────────────────────────────────┐
│            Hero 섹션                 │
│   "SaaS UI 컴포넌트를 복사해서       │
│    바로 사용하세요"                   │
│   [컴포넌트 탐색] 버튼              │
└─────────────────────────────────────┘
┌─────────────────────────────────────┐
│        인기 컴포넌트 (최대 8개)       │
│  ┌────┐ ┌────┐ ┌────┐ ┌────┐      │
│  │카드│ │카드│ │카드│ │카드│       │
│  └────┘ └────┘ └────┘ └────┘      │
│  ┌────┐ ┌────┐ ┌────┐ ┌────┐      │
│  │카드│ │카드│ │카드│ │카드│       │
│  └────┘ └────┘ └────┘ └────┘      │
└─────────────────────────────────────┘
┌─────────────────────────────────────┐
│       카테고리별 탐색 (7개)           │
│  ┌──────────┐ ┌──────────┐         │
│  │ 🧩 UI    │ │ 📊 Data  │         │
│  │Components│ │ Display  │         │
│  └──────────┘ └──────────┘  ...    │
└─────────────────────────────────────┘
┌─────────────────────────────────────┐
│             Footer                  │
└─────────────────────────────────────┘
```

### 핵심 기능

**Hero 섹션**
- 컴포넌트 총 수 배지 표시 (`45개 컴포넌트 수록`)
- `컴포넌트 탐색` 버튼 → `/explore`로 이동

**인기 컴포넌트**
- DB에서 `is_featured = true`인 컴포넌트 최대 8개 표시
- 각 카드: 정적 썸네일 + 이름 + 설명 + 태그
- 카드 클릭 → `/components/:slug` 상세 페이지로 이동

**카테고리별 탐색**
- 7개 카테고리를 아이콘 + 이름 + 설명 카드로 표시
- 카드 클릭 → `/explore/:category`로 이동

## 2. 탐색 페이지 (`/explore`)

모든 컴포넌트를 그리드로 보여주고, 필터/검색/정렬로 원하는 컴포넌트를 찾는다.

```
┌─────────────────────────────────────┐
│  모든 컴포넌트          45개의 컴포넌트 │
├─────────────────────────────────────┤
│  [All] [UI Components] [Data...]   │ ← 카테고리 필터
│                    정렬: [F] [N] [P]│ ← 정렬 버튼
├─────────────────────────────────────┤
│  ┌────┐ ┌────┐ ┌────┐ ┌────┐      │
│  │썸네│ │썸네│ │썸네│ │썸네│       │
│  │일  │ │일  │ │일  │ │일  │       │
│  │이름│ │이름│ │이름│ │이름│       │
│  │설명│ │설명│ │설명│ │설명│       │
│  │👤 ❤ │ │👤 ❤ │ │👤 ❤ │ │👤 ❤ │       │
│  └────┘ └────┘ └────┘ └────┘      │
│  ┌────┐ ┌────┐ ┌────┐ ┌────┐      │
│  │ .. │ │ .. │ │ .. │ │ .. │       │
│  └────┘ └────┘ └────┘ └────┘      │
└─────────────────────────────────────┘
```

### 핵심 기능

**카테고리 필터**
- `All` + 7개 카테고리 필 버튼
- 선택한 카테고리의 컴포넌트만 표시
- URL이 `/explore/:category`이면 해당 카테고리 자동 선택

**정렬**
- `Featured` — 추천순 (기본값)
- `Newest` — 최신순
- `Popular` — 조회수순

**검색 결과**
- URL에 `?q=검색어`가 있으면 검색 모드로 전환
- 헤더가 `"검색어" 검색 결과`로 변경
- 카테고리 필터 숨김

**컴포넌트 카드**
- 정적 SVG 썸네일 (빠른 로딩)
- 컴포넌트 이름 + 설명 (2줄 제한)
- 작성자 아바타 + 이름
- 좋아요 수 / 조회수
- 태그 (최대 3개)
- 카드 전체 클릭 → 상세 페이지로 이동

**반응형 그리드**
- 모바일: 1열 → SM: 2열 → LG: 3열 → XL: 4열

## 3. 상세 페이지 (`/components/:slug`)

선택한 컴포넌트를 실제로 동작시켜 보고, 코드를 복사한다.

```
┌─────────────────────────────────────┐
│  ← 돌아가기                          │
├─────────────────────────────────────┤
│                                     │
│         라이브 프리뷰 (전폭)          │
│    (실제 동작하는 컴포넌트 데모)       │
│                                     │
├──────────────┬──────────────────────┤
│  컴포넌트 정보  │    코드 뷰어          │
│              │                      │
│  Button      │ [Usage] [Full Source]│
│  설명 텍스트   │ ─────────── [복사]   │
│              │  import { Button }   │
│  👤 작성자    │  from './button';    │
│  📅 작성일    │                      │
│              │  export default () =>│
│  👁 조회수    │    <Button>          │
│  ❤ 좋아요    │      Click me        │
│              │    </Button>         │
│  카테고리 배지 │                      │
│  태그 목록    │                      │
│              │                      │
└──────────────┴──────────────────────┘
```

### 핵심 기능

**라이브 프리뷰**
- 페이지 상단에 전폭으로 표시
- 실제 React 컴포넌트를 렌더링 (클릭, 입력 등 인터랙션 가능)
- 45개 컴포넌트 각각에 맞는 데모 장면 구성
- 인터랙티브 컴포넌트 (Toggle, Tabs 등)는 실제 상태 변경 가능

**컴포넌트 정보 (좌측)**
- 이름, 설명
- 작성자 (아바타 + 이름 + 작성일)
- 통계 (조회수, 좋아요)
- 카테고리 배지 (클릭 → 해당 카테고리 탐색)
- 태그 전체 목록
- 프레임워크 (React)

**코드 뷰어 (우측)**
- 2개 탭 전환: `Usage Example` / `Full Source`
- 구문 강조 (prism-react-renderer, 라인 넘버 포함)
- 복사 버튼 → 클립보드에 코드 복사 → "복사됨" 피드백 2초 표시

**조회수 자동 증가**
- 페이지 진입 시 자동으로 `increment_view_count` 호출

## 데이터 흐름 요약

```
사용자 → 페이지 접속
          │
          ▼
       Zustand Store (상태 관리)
          │
          ▼
       Supabase REST API
          │
          ▼
       PostgreSQL
       ├── categories (7개 카테고리)
       └── components (45개 컴포넌트)
```

- 홈/탐색: `fetchCategories()` + `fetchComponents()` → 목록 표시
- 상세: `fetchComponentBySlug()` → 단건 조회 + `incrementViewCount()`
- 검색: `searchComponents()` → Supabase RPC (Full-text Search)
